SoftRF Alarm Log Viewer

Visualize collision alarm events on an interactive map with flight track overlay.

Overview

The Alarm Log Viewer is a web-based tool that visualizes the alarm events recorded by SoftRF during flight. It displays alarm positions on an interactive OpenStreetMap map, color-coded by severity level, with optional IGC flight track overlay.

Key features:

  • Drag-and-drop file upload for alarm logs and IGC files
  • Interactive map with alarm markers showing direction, distance, and details
  • Summary statistics: total alarms, level breakdown, time span, aircraft count
  • Sortable alarm event table
  • Clock-bearing indicators showing relative direction to threat aircraft
  • Aircraft ID lookup from database (on the hosted version)
  • Optional IGC flight track overlay with altitude at alarm time

Accessing the Viewer

The Alarm Log Viewer is hosted at:

https://ogn.helioho.st/mysoftrf/myalarms.php

It will also be available at:
https://slash-bit.github.io/SoftRF-PG/alarm-viewer.html

No Server Required The alarm viewer processes files entirely in your browser. No data is uploaded to any server. The hosted version additionally looks up aircraft IDs from a database.

Getting Your Alarm Logs

To generate alarm logs, enable the alarmlog setting (set to 1) in your SoftRF configuration.

Downloading from T-Beam

  1. Connect to the SoftRF web interface
  2. On the status page, find the "Alarm Log" section
  3. Click [Download] to save alarmlog.txt

Downloading from T-Echo

  1. Connect T-Echo to computer via USB
  2. Open the mass storage drive
  3. Copy alarmlog.txt to your computer

Using the Viewer

Uploading Files

The viewer has two upload areas:

  • Left area — Drag and drop (or click to browse) your alarmlog.txt file. This is required.
  • Right area — Optionally add an .IGC flight log file to overlay your flight track on the map.

When a file is loaded, its upload area turns green with the filename displayed.

Summary Statistics

After loading the alarm log, summary cards appear showing:

StatisticDescription
Total AlarmsNumber of alarm events in the file
Level 1 – LowCount of low-urgency alarms (≤19s to collision)
Level 2 – ImportantCount of medium-urgency alarms (≤13s)
Level 3 – UrgentCount of high-urgency alarms (≤9s)
SpanTime duration between first and last alarm
AircraftNumber of distinct aircraft IDs in the log

Map View

The map shows:

  • Green circle — Takeoff position (if recorded in the log)
  • Colored circles — Alarm positions, sized and colored by alarm level:
    • Yellow = Level 1 (Low)
    • Orange = Level 2 (Important)
    • Red = Level 3 (Urgent)
  • Clock-hand needles — Short lines showing the relative bearing to the threat aircraft
  • Numbered labels — Level 3 alarms show their sequence number
  • Dashed line — Connects alarm positions (shown when no IGC track is loaded)

Click any alarm marker for a popup with details: time, aircraft ID, direction, horizontal and vertical distance, and GPS altitude (if IGC loaded).

Alarm Table

Below the map, a scrollable table lists every alarm event with columns:

ColumnDescription
#Sequence number
Time (UTC)Time of alarm event
LevelAlarm severity (Low/Important/Urgent)
IDAircraft hex ID (with name if found in database)
DirectionRelative bearing as clock position
H Dist (m)Horizontal distance in meters
V Dist (m)Vertical distance in meters (+ = above, - = below)
Lat / LonPosition coordinates

Click any row to center the map on that alarm's position.

IGC Flight Track Overlay

Loading an IGC file adds a blue flight track line to the map. When an IGC track is present:

  • The dashed alarm path line is hidden (the flight track replaces it)
  • Alarm popups show GPS altitude at the time of each alarm
  • A legend item shows the "IGC flight track" indicator

Alarm Log File Format

The alarmlog.txt file is plain text with the following structure:

takeoff: 230415 121530 32.12345,34.56789
date,time,lat,N/S,lon,E/W,level,count,id,bearing,hdist,vdist
230415,121645,3207.407,N,03434.073,E,1,1,A8B031,14,1200,57
230415,121647,3207.412,N,03434.081,E,2,2,A8B031,12,800,43

Fields:

  • Takeoff line (optional): date, time, latitude, longitude
  • Data fields: date (DDMMYY), time (HHMMSS), latitude (NMEA format), N/S, longitude (NMEA format), E/W, alarm level (1-3), consecutive count, aircraft hex ID, relative bearing (degrees), horizontal distance (meters), vertical distance (meters)

Tips

  • Download alarm logs regularly and clear them to free space on the device
  • Pair with the corresponding IGC flight log for best visualization
  • Use the hosted version at ogn.helioho.st for aircraft ID lookup
  • Level 3 alarms deserve attention — review the pattern of encounters
  • If you see many alarms from the same ID in thermals, that's normal with nearby traffic
