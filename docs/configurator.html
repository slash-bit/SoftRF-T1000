SoftRF WebApp Configurator

How to use the web-based configurator to manage your SoftRF device.

Overview

The SoftRF WebApp Configurator is a built-in web interface available on T-Beam devices. It allows you to:

  • View device status and GNSS information
  • Change basic and advanced settings
  • Update firmware over WiFi (OTA)
  • Download and manage flight logs
  • Download alarm logs
  • Upload voice files and other data
  • Backup and restore settings
T-Beam Only The web interface is only available on T-Beam devices (which have WiFi). T-Echo devices must use USB file editing or NMEA commands instead.

A hosted version of the configurator WebApp is also available at:
https://ogn.helioho.st/mysoftrf/

Accessing the WebApp

Connecting via SoftRF WiFi

  1. Power on your T-Beam
  2. On your phone/tablet/computer, look for the WiFi network named SoftRF-xxxxxx (where xxxxxx is the device ID)
  3. Connect using password 12345678
  4. Your device may warn "no internet access" — that's correct, allow it
  5. Open a web browser and go to http://192.168.1.1
Custom Network Name You can customize the WiFi network name using the myssid setting. The password remains 12345678 and the IP is always 192.168.1.1.

Using an External WiFi Network

If you configure SoftRF to connect to an existing WiFi network (via the ssid and psk settings):

  1. SoftRF tries to connect for 10 seconds after boot
  2. If successful, connect your phone/computer to the same network
  3. Check the OLED display (4th or 5th page) for the assigned IP address
  4. Browse to that IP address

If the connection fails, SoftRF falls back to creating its own network.

Status Page

The home page shows:

  • Firmware version — current installed version
  • Device ID — unique hardware identifier
  • Aircraft ID — transmitted ID and type
  • GNSS status — fix quality, satellite count, leap seconds
  • Altitude — both ellipsoid and MSL, plus geoid separation
  • Battery voltage — current charge level
  • Radio info — protocol, band, packets sent/received, max RSSI
  • WiFi/Bluetooth status
  • Settings status — whether settings were preserved or reset

Action buttons on the status page:

  • Basic Settings — essential configuration
  • Advanced Settings — all settings with raw codes
  • Firmware Update — OTA firmware upload
  • Landed-Out Mode — activate relay mode for landed aircraft
  • Flight log buttons — download, list, clear
  • Alarm log buttons — download, clear
  • File management — upload, manage SPIFFS files
  • Reset GNSS — factory-reset GNSS module and cold-start

Basic Settings Page

The basic settings page (since MB153) shows a shorter, cleaner form with the most commonly needed settings:

  • Mode (Normal, GNSS Bridge)
  • Aircraft ID and ID type
  • Protocol, Region, Aircraft type
  • Alarm trigger method
  • Volume (buzzer)
  • Bluetooth mode
  • NMEA output destinations and sentence types
  • Flight logging options
  • Stealth and No-track flags

After changing settings, scroll to the bottom and click "Save and Reboot". Reconnect to WiFi afterward and verify your settings took effect.

Backup Note Clicking "Save and Reboot" automatically backs up the previous settings file. You can revert by clicking "Restore/Swap" on the status page.

Advanced Settings Page

The advanced settings page shows all settings with their raw numeric codes and comments explaining values. You can enter codes directly. See the Under the Hood page for code reference.

Firmware Update

  1. Download the latest .bin firmware (~1.7 MB) from GitHub
  2. Ensure the battery is fully charged
  3. Click "Firmware Update" on the status page
  4. Choose the .bin file and click "Update"
  5. Wait for "UPDATE DONE, REBOOTING" (may take over a minute)
  6. The OLED counter increments during upload; blue LED flashes once/second
  7. Do not touch anything for a full minute after completion
WiFi Interference If a stronger external WiFi network is nearby, your device may switch networks mid-update, causing failure. Move away from external networks or configure SoftRF to use the external network.

File Management

Flight Logs

The status page shows buttons depending on the logging destination:

  • RAM/Flash logging: "Download Current", "List Archived", "Clear Archive"
  • SD card logging: "Download Latest", "List Files", "Clear Files"

Compressed logs (.IGZ) on T-Beam are automatically decompressed when downloaded through the web interface.

Alarm Log

If alarm logging is enabled (alarmlog=1), each collision alarm event is recorded to alarmlog.txt. Use the "Download" button to retrieve it, or "Clear" to start fresh. View the log visually with the Alarm Log Viewer.

Settings Backup and Restore

  • Download — Save settings.txt to your computer
  • Upload — Upload an edited settings.txt (takes effect after reboot)
  • Backup — Copy current settings to settingb.txt internally
  • Restore/Swap — Swap settings.txt with settingb.txt and reboot

Voice Files

Upload the waves.tar file (17 WAV files, ~150 KB total) using the upload button at the bottom of the status page. The status page should then show "17 WAV files found".

Landed-Out Mode

Activate via the button on the status page, or via a long-press of the middle button on T-Beam. In this mode, the aircraft type is set to zero, marking the device for relay by airborne aircraft. This extends the range at which the landed aircraft can be seen.

Troubleshooting

Cannot access the web interface

  • Ensure you're connected to the correct WiFi network
  • Try the IP address 192.168.1.1 (for SoftRF's own network)
  • If Bluetooth is active, the web interface may be disabled due to memory — a notice will appear
  • Try recovery via USB terminal and NMEA config commands

Settings were reset after firmware update

This happens when changing between incompatible settings formats (e.g., mainline to Moshe's fork). Both the OLED and web page warn when this occurs. Download and keep backups of your settings.txt file.

Firmware update failed

  • Try again ensuring stable WiFi connection
  • Do not start from "RC9" version via WiFi — use USB method
  • If the device won't boot, use the USB flashing method (ESP tool or web-based flasher)

Hosted WebApp

A hosted version of the SoftRF configurator interface is available at:

https://ogn.helioho.st/mysoftrf/

This hosted version can be used for:

  • Browsing the configurator interface without a device connected
  • Familiarizing yourself with the available settings before configuring your device
  • Reference for settings values and their meanings
Note The hosted WebApp cannot directly communicate with your SoftRF device. To change settings on your device, use the device's own web interface as described above.
